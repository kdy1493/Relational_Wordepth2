# Relational Ranking Loss for Monocular Depth Estimation — 전체 정리

---

## 1. 내가 하고 있는 것 (한 줄 요약)

**기존 monocular depth 모델들이 "어떤 객체가 앞이고 뒤인지"를 자주 틀리는 문제를, 모델 구조를 건드리지 않고 loss만 추가해서 고치는 연구.**

---

## 2. 핵심 아이디어

| 항목 | 내용 |
|------|------|
| **문제** | 기존 모델은 pixel-wise loss만 쓰기 때문에, 평균 깊이 오차(AbsRel)는 좋아도 객체 간 앞/뒤 순서가 자주 뒤집힘 |
| **해결** | GT depth + instance segmentation으로 객체 쌍의 앞/뒤 관계를 추출하고, 예측 depth에서도 그 순서가 유지되도록 hinge loss(Relational Ranking Loss)를 추가 |
| **차별점** | (1) pixel-pair가 아니라 object-instance pair 단위 (2) 대표값에 μ + ασ (statistical representative) 사용 (3) 모델 구조 변경 없이 plug-in으로 적용 |
| **목표 학회** | ACM MM (또는 유사 수준) |

---

## 3. 수식 요약

```
L_total = L_base + λ_rel × L_rel

객체 대표값:  d_k = μ_k + α × σ_k
관계 벌점:    v(A,B) = max(0, d_A - d_B + m)
전체 관계 loss: L_rel = (1/|R_valid|) × Σ w × v(A,B)

RSR (평가 지표) = 관계를 만족하는 쌍의 수 / 전체 유효 관계 쌍의 수
```

**손잡이 3개:** λ_rel (loss 강도), m (margin), α (불확실도 반영 정도)

---

## 4. 논문의 Contribution (3개)

1. **Relational Ranking Loss** — 객체 간 상대적 깊이 순서를 학습 제약으로 부과하는 plug-in loss
2. **RSR (Relation Satisfaction Rate)** — 기존 pixel metric이 못 잡는 구조적 정합성을 측정하는 새로운 평가 지표
3. **실험적 검증** — 4개 모델(DORN, AdaBins, Baseline, WorDepth)에서 model-agnostic하게 개선됨을 확인

---

## 5. 실험 6개 — 각각 뭘 증명하는지

### 실험 1: Motivation Table (기존 모델들이 관계를 얼마나 틀리나)

| 목적 | 기존 모델들의 RSR이 낮다는 걸 숫자로 보여줘서 "이 문제가 진짜 있다"를 증명 |
|------|------|
| 방법 | DORN, AdaBins, Baseline, WorDepth의 pretrained 모델 → test set에서 RSR 측정 |
| 기대 결과 | RSR이 0.7~0.8대 → "20~30% 관계가 틀린다" |
| 논문 위치 | Introduction 또는 Experiment 서두 |

### 실험 2: Main Table (Base vs +Ours, 4개 모델)

| 목적 | Relational Loss를 추가하면 RSR이 올라가고, 기존 metric은 유지됨을 증명 |
|------|------|
| 방법 | 4개 모델 × {Base, +Ours} = 8개 학습, 고정 세팅: α=1.0, λ=0.1, m=0.1 |
| 기대 결과 | AbsRel/RMSE 유지 또는 소폭 개선 + RSR 확실히 상승 (4개 모델 전부) |
| 논문 위치 | Main Results (메인 테이블) |

### 실험 3: vs 기존 Ordinal Loss (왜 object-level이 더 좋은가)

| 목적 | pixel-pair ordinal loss보다 우리 방법이 RSR에서 더 효과적임을 증명 |
|------|------|
| 방법 | WorDepth 기준, 동일 λ/학습 조건에서 loss만 교체: Base / +Pixel-Pair Ordinal / +Ours |
| 기대 결과 | Pixel-pair도 RSR 약간 상승, 하지만 Ours가 더 많이 상승 |
| 논문 위치 | Comparison with Existing Methods |

### 실험 4: α Ablation (Statistical Representative 정당화)

| 목적 | σ를 반영한 대표값(μ + ασ)이 단순 평균(μ)보다 낫다는 걸 증명 |
|------|------|
| 방법 | WorDepth 기준, λ=0.1, m=0.1 고정, α = {0, 0.5, 1.0} |
| 기대 결과 | α=1.0이 RSR에서 가장 높음 |
| 논문 위치 | Ablation Study |

### 실험 5: λ_rel Sweep (Loss 강도 Sweet Spot)

| 목적 | "왜 λ=0.1을 선택했는가"에 대한 근거 |
|------|------|
| 방법 | WorDepth 기준, α=1.0, m=0.1 고정, λ = {0.05, 0.1, 0.2} |
| 기대 결과 | 0.05는 효과 약함, 0.2는 AbsRel 약간 나빠짐, 0.1이 sweet spot |
| 논문 위치 | Ablation Study |

### 실험 6: Qualitative (시각적 증거)

| 목적 | 사람 눈으로 봐도 구조가 좋아졌다는 걸 보여줌 |
|------|------|
| 방법 | violation Top-K 샘플에서 Base vs +Ours depth map 나란히 비교, 객체 쌍 관계 표시 |
| 기대 결과 | Base는 관계 위반(❌), +Ours는 관계 만족(✅) |
| 논문 위치 | Qualitative Results |

---

## 6. RSR 정당화 (이것도 해야 함)

RSR은 내가 새로 만든 metric이기 때문에, "이 숫자가 의미 있다"를 증명해야 함.

| 해야 할 것 | 내용 |
|------|------|
| **Scatter plot** | 이미지별 AbsRel vs RSR 찍기 → 상관이 약하면 "기존 metric이 못 잡는 걸 RSR이 잡는다" 증명 |
| **RSR 고/저 시각화** | RSR 높은 이미지 vs 낮은 이미지 나란히 → RSR이 실제 구조 오류와 대응됨을 보여줌 |
| **(선택) m robustness** | m을 여러 값으로 바꿔도 +Ours > Base 관계가 유지됨을 supplementary에 보고 |

## 9. 리뷰어 예상 공격과 방어

| 예상 질문 | 방어 |
|------|------|
| "그냥 hinge loss를 object level로 올린 거 아니냐?" | statistical representative(μ+ασ)의 ablation 결과 + pixel-pair 대비 우위 실험 |
| "자기 metric(RSR)으로 자기 평가하면 당연히 좋은 거 아니냐?" | AbsRel vs RSR scatter plot (약한 상관 → 기존 metric이 못 잡는 걸 잡음) + qualitative |
| "m이나 α 바꾸면 결과 뒤집히는 거 아니냐?" | m robustness 표 (supplementary) + α ablation |
| "특정 모델에서만 되는 거 아니냐?" | 4개 모델 cross-model 실험 |
| "segmentation 품질에 의존하는 거 아니냐?" | 어떤 segmentation 모델을 썼는지 명시 + segmentation 에러 분석 (가능하면) |
| "downstream에서 실제로 도움이 되냐?" | (가능하면) point cloud 시각화 또는 간단한 downstream 실험 추가 |
